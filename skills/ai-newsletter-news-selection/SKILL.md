## 1. 产出与筛选目标

- **产品**：CEO 与创始人专读 AI Newsletter（高风），面向 C-suite。
- **筛选目标**：从候选新闻中选出 8–12 条，每栏 2–3 条，能贡献**三条曲线**的增量信号并导出管理动作。
- **三条曲线**（仅用于筛选与打分，不写入正文）：
  - **技术曲线**：能力边界/成本结构/接口生态/产品形态变化。
  - **组织曲线**：治理、审计、合规、跨部门协同、责任归属。
  - **人才曲线**：岗位重构、评价标准、能力认证、培训机制。

---

## 2. 四个栏目（用于分类）

| 栏目 | 每期条数 |
|------|----------|
| 大模型竞技场 | 2–3 条 |
| AI产品探新 | 2–3 条 |
| 企业AI前沿与AI原生企业 | 2–3 条 |
| 先行者观点 | 2–3 条 |

- 选取顺序：**硬性门槛 → 分类 → 打分**。
- 时效：发布时间须在「今天」起算 **7 天内**。

### 2.1 曲线信号定义（用于打分）

- **技术曲线信号**：模型能力、定价/成本、API/生态、Copilot→Agent 等可核对变化。
- **组织曲线信号**：权限/审计/合规、跨部门协同、AI CoE、供应风险等。
- **人才曲线信号**：角色重构、评价标准、AI literacy、训练营等。
- 同等条件下：**组织 > 人才 > 技术**。

## 3. 采集（Crawler）

### 3.1 运行方式

```bash
pip install -r scripts/requirements-crawler.txt
python scripts/news-crawler.py --output crawler-output/candidates.json --days 7
```

**输出**：`crawler-output/candidates.json`，含 `title`、`link`、`published`、`source`、`snippet`。

**X/Twitter**：需 `TWITTER_BEARER_TOKEN`；仅拉取列表内账号（`X_HANDLES`），**只保留热度最高的 10 条**（点赞+转发+回复）。

### 3.2 来源

| 类型 | 说明 |
|------|------|
| X (Twitter) | 列表内账号，热度前 10 条 |
| RSS/博客 | Lenny、Anthropic、OpenAI、DeepMind 等 |
| YouTube | Karpathy、Lex Fridman、MattVidPro AI 等 |
| 微信公众号 | 机器之心、量子位等，经 wechat-article-exporter 导出后补充 |

完整列表见 content-sources.md。

### 3.3 采集流程

1. **必须运行 crawler**。
2. 读取 `crawler-output/candidates.json`。
3. 视需补充 web 检索、微信公众号；保留具体来源 URL。
4. **禁止**未运行 crawler 时仅用 web search 生成整期。

## 4. 筛选流程（必须按顺序执行）

### 4.1 硬性门槛（任一不满足直接淘汰）

- **事实**：须有具体文章/页面 URL；时间、主体、事件、数字可核对。
- **主体**：新闻主体明确。
- **时效**：7 天内发布。
- **不重复**：同一事件只保留 1 条；全期不跨栏重复。
- **可落地**：能导出管理动作（治理/流程/组织/指标/生态/人才）。
- **禁区**：纯跑分/纯融资/纯营销软文且无管理动作 → 淘汰。

### 4.2 分类（单独一步）

对通过门槛的每条候选归入四类之一：

| 类别 | 对应栏目 |
|------|----------|
| 大模型 | 大模型竞技场 |
| 产品 | AI产品探新 |
| 企业应用 | 企业AI前沿与AI原生企业 |
| 大V/观点 | 先行者观点 |

- 规则：Karpathy、Lenny 访谈、CEO/CTO 表态 → 大V/观点。
- 分类不写入正文。

### 4.3 打分（确定性公式）

**总分（1–5）= round( 0.25×栏目相关性 + 0.30×边界/规则变化 + 0.35×企业可操作性 + 0.10×风险/约束 , 1位小数 )**

**四维度锚点**：

- **栏目相关性**：5=高度匹配；3=可归类有偏移；1=勉强。
- **边界/规则变化**：5=明确边界/规则变化；3=趋势但缺硬边界；1=无。
- **企业可操作性**：5=能导出 2–3 个可验收动作；3=方向性；1=无法落地。
- **风险/约束**：5=明确约束与规避；3=模糊；1=无。

**自动淘汰**：企业可操作性 ≤2 或边界/规则 ≤2 且无管理动作 → 通常淘汰。

### 4.4 入选规则

- 每栏 2–3 条，总计 8–12 条。
- X 来源：crawler 已限制 10 条；每期从 X 最多选 2–3 条写入正文。
- Tie-breaker：官方/一手/权威 > 边界变化 > 可操作性 > 新鲜度 > 组织/人才曲线覆盖。

### 4.5 输出格式

**入选（按四栏分组）：**
【标题｜发布日期｜来源URL｜分类（大模型/产品/企业应用/大V观点）｜入选理由（曲线增量）｜总分】

**淘汰：**
【标题｜淘汰原因】

## 5. 筛选说明（必须产出，供用户审阅）

筛选完成后，**必须**输出以下说明，告知用户筛选理由与评分依据：

### 5.1 入选条目的评分明细

对每条**入选**新闻，逐条给出：

| 条目 | 栏目相关性 | 边界/规则变化 | 企业可操作性 | 风险/约束 | 总分 | 说明 |
|------|------------|---------------|--------------|-----------|------|------|
| 标题 | 1–5 分 | 1–5 分 | 1–5 分 | 1–5 分 | 计算值 | 为何入选、贡献的曲线信号 |

- 四维度各给分及简要理由（一句话）。
- 总分按公式计算：`round(0.25×A + 0.30×B + 0.35×C + 0.10×D, 1)`。

### 5.2 淘汰条目的淘汰理由

对每条**淘汰**新闻，说明：

- **若为门槛淘汰**：命中哪条（事实/主体/时效/不重复/可落地/禁区）。
- **若为打分淘汰**：四维度得分与总分；为何未达入选线或同栏被挤掉。

### 5.3 整体筛选理由

用一段话说明：

- 本期为何选这 8–12 条（栏目平衡、曲线覆盖、来源多样性等）。
- 同栏有多条候选时，为何保留这几条、淘汰其它（Tie-breaker 应用）。
