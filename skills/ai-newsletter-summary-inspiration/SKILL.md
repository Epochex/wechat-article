## 1. 每条新闻的固定结构

**读者可见正文**须包含且仅包含三项：

| 要素 | 要求 |
|------|------|
| **一段描述** | 100–150 字，含时间、主体、事件；与来源一致，不夸大。 |
| **来源** | 直接写 URL，不写括号、注释或链接文字。 |
| **一条启发（管理层建议）** | 150–200 字，须符合 §2 约束。 |

---

## 2. 启发（Inspiration）撰写规范

### 2.1 启发生成的硬规则（必选）

- **先结论后证据**：首句必须是「趋势/判断」，不是新闻复述。
- **必须完成问题重述**：把问题从「模型/功能/工具」转成「组织/流程/治理/验收/主权」。
- **必须有对照**：写出「错误路径 vs 正确路径」，并点明代价/风险。
- **必须落到机制动作**：至少给出 2–3 个「可落地机制」，从以下列表中挑选：
  - 权限分层 / 日志留痕 / 审计追溯 / 回滚机制
  - 目标–约束–验收（Definition of Done）
  - 内部评测任务集/考题（业务真实基线）
  - 知识拆解→模板/SOP→Skill 化/可调用资产
  - 生态绑定与可切换路径（降低断供/抬价风险）
- **必须回扣三条曲线**：明确属于**技术/人才/组织**哪条线，并说明这条线的「斜率变化」。
- **语言约束**：面向传统企业管理者，避免互联网黑话；允许一个强类比，但不要堆比喻。
- **不写具体周期**：禁止 Q2、本季度、下半年、明年 Q1；可用「形成」「定期」「在规划时」。

### 2.2 启发段落的模板（强烈建议固定）

每段启发须自然包含以下五部分（可不写小标题，但逻辑顺序固定）：

1. **一句话判断（趋势 + 换挡）**：先结论后证据。
2. **为什么（用新闻作证据，但不复述细节）**：扼要点明来源依据。
3. **对企业的真正影响（组织视角）**：成本/替代/边界/责任。
4. **管理动作清单（机制级 2–3 条）**：从 §2.1 机制列表中选。
5. **风险提示（如果不做，会以什么方式失控）**：错误路径与代价。

### 2.3 字数与呈现

- **150–200 字**，简体中文，面向 C-suite。
- 呈现形式：**一段完整连贯文字**，禁止拆成小标题或 bullet；内部逻辑按 §2.2 五部分自然展开。

---

## 3. 反幻觉规则

### 3.1 先有来源再写内容

- 禁止：无可验证链接就写新闻或定性结论。
- 正确：根据具体链接撰写描述与启发；不得编造或使用占位链接。

### 3.2 事实必须有来源

- 时间、主体、事件、数据须在来源中有对应表述。
- 产品/模型/公司/人名与官方一致。

### 3.3 禁止的表述（除非原文明确且附出处）

- 无来源定性词：冲击/颠覆/革命/首家/最大/全面超越。
- 无来源数据：市占率、下载量、股价涨跌幅等。

### 3.4 合规

- 单源连续引用不超过 30 字；改写时加说明：「内容经改写以符合授权限制。」

---

## 4. 文件与输出格式

- **命名**：主版本 `第[N]期`，迭代版本 `第[N]版`。
- **格式**：Markdown（草稿/编辑）；Word（审阅）；PDF（终版）。
- 链接可访问、去除追踪参数。

---

## 5. 编审检查清单

撰写或修改 newsletter 时逐条核对：

- [ ] 每条新闻有**描述 + 来源 URL + 启发**，描述与链接一致。
- [ ] 启发**首句是趋势/判断**，非新闻复述；**先结论后证据**。
- [ ] 启发完成**问题重述**：从模型/功能/工具转为组织/流程/治理/验收/主权。
- [ ] 启发有**对照**：错误路径 vs 正确路径，点明代价/风险。
- [ ] 启发含**2–3 个可落地机制**，从 §2.1 列表中选；并给出**风险提示**（不做会如何失控）。
- [ ] 启发**回扣三条曲线**：明确技术/人才/组织哪条线，说明斜率变化。
- [ ] 启发为一段完整文字，150–200 字；不写 Q2 等具体周期。
- [ ] 无未附来源的「冲击」「颠覆」「革命」；产品/模型名与官方一致。
- [ ] 日期、数字与来源一致；无虚构或混淆。
- [ ] 来源仅写 URL，无括号、注释或链接文字。
- [ ] 面向传统企业管理者，避免互联网黑话；比喻不超过一个。
